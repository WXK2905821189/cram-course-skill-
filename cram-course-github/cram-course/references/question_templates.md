# 大学期末考试 — 题型模板与出题指南

> [!warning]
> 题目、答案、解析、分值和考频判断由 AI 根据材料生成，必须结合教材原文、老师范围、历年题或标准答案进行人工复核。

## 出题通用原则

1. **考点优先**：每道题必须对应一个明确的知识点，不出现"你知道就知道"的偏题
2. **难度分层**：基础题 60% + 中档题 30% + 拔高题 10%，模拟期末考真实分布
3. **解析到位**：每题附带答案解析——不仅说"为什么对"，还要说"其他选项为什么错"
4. **反流水线**：选项不出现明显的"三长一短""选项C永远正确"等规律

## 题型一：选择题

### 适用场景
概念辨析、公式识别、流程排序、正误判断

### 模板

```markdown
**Q1.** [题干——简洁明确，不含歧义]

A. [选项A]
B. [选项B]
C. [选项C]
D. [选项D]

> **答案**：X
> **解析**：[为什么选X，以及为什么不选其他 —— 1-2句话]
> **考点**：[对应知识点标签]
```

### 示例

```markdown
**Q1.** 栈的特点是？

A. 先进先出
B. 先进后出
C. 随机存取
D. 按索引存取

> **答案**：B
> **解析**：栈是后进先出（LIFO）结构，只能在栈顶进行插入和删除。A 是队列的特点，C/D 是数组的特点。
> **考点**：栈的基本概念
```

### 出题技巧
- 错误选项应该来自常见误区（学生容易犯的错）
- 选项之间要有相关性和迷惑性，不要出现明显无关的选项
- 选项长度尽量一致，避免暗示

## 题型二：填空题

### 适用场景
关键定义、公式参数、重要数字、专业术语

### 模板

```markdown
**Q1.** [题干，留空处用 `_____` 表示]

> **答案**：[填写内容]
> **解析**：[1句话说明为什么]
> **考点**：[对应知识点标签]
```

### 示例

```markdown
**Q1.** 二叉搜索树的中序遍历结果是 _____ 序列。

> **答案**：递增有序
> **解析**：二叉搜索树定义：左子树 < 根 < 右子树，中序遍历先左后根再右，因此结果递增有序。
> **考点**：二叉搜索树性质
```

### 出题技巧
- 一题只填一个空，避免多空导致的连锁错误
- 填空处应是核心关键词/数字，而非辅助词
- 提供清晰上下文，让答案唯一确定

## 题型三：判断题

### 适用场景
常见误区、概念边界、公式适用条件

### 模板

```markdown
**Q1.** [陈述句]

> **答案**：[正确/错误]
> **解析**：[如正确，简述理由；如错误，指出错在哪里并给出正确说法]
> **考点**：[对应知识点标签]
```

### 示例

```markdown
**Q1.** 所有递归算法都可以转化为非递归算法。

> **答案**：正确
> **解析**：递归的本质是系统栈，任何递归都可以通过显式使用栈来模拟，因此可以转化为迭代形式。
> **考点**：递归与迭代的关系
```

### 出题技巧
- "总是"、"所有"、"一定"等绝对化表述通常是错误选项（但不要机械套用）
- 将"容易混淆的概念 B 的特征"伪造成概念 A 的判断句，制造迷惑
- 错误选项常见套路：偷换概念、范围错误、条件缺失

## 题型四：简答题

### 适用场景
核心理论阐述、流程描述、概念对比、公式推导

### 模板

```markdown
**Q1.** [问句 —— 明确、聚焦，不要求长篇大论]

> **参考答案**：[分点作答，3-5 个要点即可]
> **评分要点**：
> 1. [要点1 — 分值分配]
> 2. [要点2 — 分值分配]
> ...
> **考点**：[对应知识点标签]
```

### 示例

```markdown
**Q1.** 简述快速排序的基本思想及其时间复杂度。

> **参考答案**：
> 1. 选取一个基准元素（pivot）
> 2. 将数组分为两部分：小于基准的放左边，大于基准的放右边
> 3. 对左右两部分递归执行上述过程
> 4. 平均时间复杂度 O(n log n)，最坏 O(n²)（当每次选到最值作为基准时）
> **评分要点**：
> 1. 选基准 + 分区过程 — 3 分
> 2. 递归描述 — 2 分
> 3. 时间复杂度 + 最坏情况 — 2 分
> **考点**：快速排序算法
```

### 出题技巧
- 题目要对焦在"可列举"的内容上（特点、步骤、区别、条件），避开"漫谈式"问题
- 评分要点帮助用户自查"是否答全了"，也帮助理解考试得分点

## 背题模式格式

快速复习时使用紧凑格式，去掉详细解析只保留答案关键词：

```markdown
## 第 X 章 背题速过

1. 栈的特点是？ → **先进后出 (LIFO)**
2. 队列的特点是？ → **先进先出 (FIFO)**
3. 二叉树第 k 层最多几个节点？ → **2^(k-1)**
4. 快排平均时间复杂度？ → **O(n log n)**
5. 归并排序是稳定排序吗？ → **是**
```

## 综合模拟卷结构

按以下比例组卷（以 100 分制为例）：

| 题型 | 题量 | 分值 | 来源 |
|------|------|------|------|
| 选择题 | 20题 | 40分 | 各章均匀分布 |
| 填空题 | 10题 | 20分 | 核心概念为主 |
| 判断题 | 10题 | 10分 | 易混淆点为主 |
| 简答题 | 5题 | 30分 | 重点章节各1题 |
| **总分** | **45题** | **100分** | — |

模拟卷后附答案 + 每题对应知识点标签，方便定位薄弱环节。

## 专项题型训练（跨章整合）

详见 `assets/topic_drill_template.md`。

在逐章学习完成后，必须生成以下 5 个专项训练文件：

| 专项文件 | 内容 | 目的 |
|---------|------|------|
| `30_名词解释专项.md` | 全课程所有定义汇总，分组编排 | 名词解释题一网打尽 |
| `31_公式计算专项.md` | 全课程所有公式 + 计算题，按难度递进 | 避免考场上公式记混 |
| `32_对比辨析专项.md` | 全课程所有易混淆概念对比 + 辨析题 | 攻克"选 A 还是选 B" |
| `33_简答论述专项.md` | 全课程所有简答/论述题，按考频排序 | 大题不丢分 |
| `34_综合应用专项.md` | 跨章节综合应用题 | 应对期末"大题" |

### 专项训练的命题原则

1. **去重**：同一知识点在逐章题库中已出现，专项训练中换角度/换问法
2. **升维**：专项训练比逐章题库更难——公式专项要有变形，对比专项要有"选正确说法"的综合选择题
3. **全覆盖**：专项训练中每个知识点至少出现一次，确保不遗漏
4. **考试导向**：专项训练的结构模拟期末考卷的题型分布
